# PDF Generation Layer

This layer converts LLM-generated packet content into branded, downloadable PDFs. It sits between the LLM output (stored as Markdown in `generated_packets.output_markdown`) and the user-facing download experience.

## What the PDFs include

Each generated PDF is a complete proposal packet containing:

- **ElevateUT logo and branding** — brand colors, tagline, professional layout
- **Tailored one-pager** — customized for the specific BLM office, explaining why disc golf is a good fit for their managed lands
- **EXPLORE Act alignment memo** — references specific provisions of the EXPLORE Act that support recreation expansion on public lands
- **Cover letter** — addressed to the office's recreation planner (when known), introducing ElevateUT and the proposal
- **Suggested contacts** — local partners, elected officials, and BLM personnel to engage

## Technology choice (TBD)

The PDF rendering library has not been selected yet. The three candidates are:

| Library | Approach | Weight | Pros | Cons |
|---------|----------|--------|------|------|
| **puppeteer** | Renders full HTML/CSS to PDF via headless Chromium | Heavy (~300 MB) | Reuses Tailwind/DaisyUI styles from the site; pixel-perfect output | Large dependency; slow cold starts; needs Chromium binary in deployment |
| **@react-pdf/renderer** | Builds PDF from a React component tree using its own layout engine | Medium (~15 MB) | Clean component model; no browser needed; good text/layout support | Requires a parallel set of React components; cannot reuse Tailwind classes directly |
| **pdfkit** | Programmatic PDF construction (draw text, shapes, images) | Light (~5 MB) | Smallest dependency; fast; full control over every element | Most manual layout work; no HTML/CSS; harder to maintain visual consistency with site |

Decision criteria: deployment target (Cloudflare Workers rules out puppeteer), design fidelity requirements, and maintenance burden.

## Storage and linking

Generated PDFs can be stored in **Supabase Storage** (bucket: `packets`). The public URL is then saved in the `generated_packets.output_pdf_url` column so the packet can be re-downloaded without regeneration.

Flow:
1. User clicks "Build My Packet" on an office detail page
2. Server checks `generated_packets` for a cached recent packet
3. If none exists (or it's stale), the LLM layer generates new Markdown content
4. This PDF layer renders the Markdown into a branded PDF Buffer
5. The Buffer is uploaded to Supabase Storage
6. The storage URL is saved back to `generated_packets.output_pdf_url`
7. The Buffer is returned to the user as a file download

## Branding guidelines

- **Colors**: Terra Cotta `#B85C38`, Sage `#5B7F3B`, Summit Gold `#D4952B`, Night Sky `#1E2D3B`, Sandstone `#F5F0E8`
- **Tagline**: "Disc golf on America's public lands." / "Where the wild things fly."
- **Tone**: Professional but not corporate. Approachable, community-oriented, data-driven.
- **Logo**: Placed in the PDF header. Source files at `public/images/brand/`.
- **Footer**: Page numbers, generation date, "Generated by EXPLORE Disc Golf — explorediscgolf.org"
